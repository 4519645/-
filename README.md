# 作业 3：汽车性能与产地交互式可视化探索

**项目部署地址 (Live Demo): https://4519645.github.io/-/

## 项目简介
本项目是“作业3-交互式可视化”的最终提交成果。该项目基于 `cars.json` 数据集，利用 Vega-Lite 构建了一个无需服务器端支持的纯前端交互式数据可视化 Web 应用。项目旨在通过多种交互技术，帮助用户探索不同产地汽车在马力、油耗及气缸数等维度的关联与分布。

---

## 报告内容 (Assignment Report)

### 1. 你的可视化要回答的问题是什么？
此可视化作品主要致力于帮助用户探索和回答以下问题：
* **核心问题：** 不同产地（美国、欧洲、日本）的汽车在**引擎动力（马力）**和**燃油经济性（每加仑英里数）**之间呈现怎样的关系差异？
* **衍生问题：** 当我锁定某一特定性能区间（例如：低马力高油耗的环保车型）时，这批车辆的**平均气缸数**特征是什么？各产地在这一区间内的结构分布情况如何？

### 2. 设计决策的原理
为了清晰地回答上述问题，我进行了以下视觉编码和交互设计决策：

* **视觉编码 (Visual Encodings):**
  * **主视图（散点图）：** 因为需要展示“马力”与“油耗”两个连续型变量的相关性，散点图是最佳选择。
  * **副视图（柱状图）：** 用于展示和对比类别数据（产地）的聚合数值（平均气缸数），柱状图具有直观的比较优势。
  * **颜色通道：** 两个视图均使用颜色来编码“产地 (Origin)”这一分类变量。这在视觉上连接了两个分离的图表，减少了用户的认知负担。
* **交互技术 (Interactive Techniques):**
  * **按需详情 (Tooltip)：** 悬停显示具体车型、年份等信息，避免将文字直接扔在图表上造成视觉混乱。
  * **平移与缩放 (Pan & Zoom)：** 散点图数据点较密集，缩放功能便于观察局部簇状分布 (Clusters) 和离群值。
  * **刷选与多视图联动 (Brushing & Cross-filtering)：** 左图的矩形框选直接作为右侧柱状图的过滤器（动态查询）。这使得分析能够从宏观走向微观。
  * **多维演变分析： 新增了时间序列折线图，并实现了与散点图的双向联动。用户可以通过刷选散点观察特定性能群体的技术演变趋势，也可以通过点击时间轴溯源特定年份的车辆分布。
  * **图例点击高亮 (Legend Binding)：** 允许通过点击图例将特定产地的汽车“凸显”出来（其他产地降低透明度），同时联动右侧统计图计算。
* **替代方案与最终选择 (Alternative Considered):**
  * 最初考虑过使用气泡图（将气缸数映射为散点的大小），但在实际测试中发现，散点重叠严重导致大小难以分辨（视觉编码失效），且显得杂乱。最终决定将“气缸数”剥离出来，用一个联动的柱状图单独展示。事实证明，“焦点+上下文”的多视图协同极大地提升了探索体验。

### 3. 外部资源引用
* **数据集：** 本项目直接通过 Web API 请求了 Vega-datasets 官方提供的公共数据集 [`cars.json`](https://raw.githubusercontent.com/vega/vega-datasets/master/data/cars.json)。
* **依赖库：** 项目使用了 [Vega](https://vega.github.io/vega/) (v5), [Vega-Lite](https://vega.github.io/vega-lite/) (v5) 以及 [Vega-Embed](https://github.com/vega/vega-embed) (v6) 进行图表渲染和解析。

### 4. 开发过程概述
* **团队分工：** 本项目由赵李源独立完成。
* **最耗时的方面：**
  * 开发中最具挑战性也是耗时最长的部分在于**“复杂交互逻辑的绑定与多视图状态同步”**。起初，使用 `brush` 实现框选联动比较顺利，但在实现“点击图例高亮”时遇到了冲突。为了让图例过滤（`leg_sel`）不仅能改变散点图的透明度（opacity），还要能完美同步过滤右侧的柱状图数据，查阅了大量 Vega-Lite 文档中关于 `condition` 和 `transform/filter` 数组配置的资料，经过多次调试才实现平滑的联动效果。

## 本地运行指南
本项目为纯静态前端页面，无需配置 Node.js 环或数据库。
1. 下载/克隆本仓库到本地。
2. 双击打开 `index.html`，系统将默认调用浏览器（推荐使用 Chrome 或 Edge）渲染可视化。
3. 或使用 VS Code 的 Live Server 插件打开以获得更好的本地预览体验。
