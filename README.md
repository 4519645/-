# 实验报告：汽车工业演变交互式分析系统 (作业 3)

**项目部署地址 (Live Demo):** https://4519645.github.io/-/

## 1. 项目综述
本项目基于经典的 `cars.json` 数据集，利用 **Vega-Lite** 构建了一个多视图协同的交互式探索系统。系统通过散点图、聚合柱状图和时间趋势图的联动，旨在揭示 1970-1982 年间，不同产地汽车在动力性能、燃油经济性以及技术演进路径上的差异。

---

## 2. 核心问题探索
本可视化旨在回答以下三个层层递进的问题：
1.  **性能相关性：** 马力（Horsepower）与燃油效率（MPG）是否存在普适的负相关性？不同产地的车型在这一分布中有何聚类特征？
2.  **技术结构：** 当我们锁定“高效率车型”（高 MPG 区域）时，这些车辆在气缸数（Cylinders）这一核心硬件指标上呈现出怎样的产地特征？
3.  **时间演变：** 随着全球石油危机和技术进步，各产地的平均燃油效率随时间是如何演变的？在特定时间点（如 1973 年石油危机后），各产地的响应速度有何差异？

---

## 3. 设计决策与视觉编码

### 3.1 视图构成 (View Composition)
*   **主视图 (散点图)：** 采用 $X$-$Y$ 轴映射连续变量，通过颜色编码产地。支持缩放和平移以观察数据密集区的离群值。
*   **侧视图 (统计柱状图)：** 使用聚合变换（Mean）展示平均气缸数。作为主视图的“细节辅助”，实时反馈框选区域的硬件特征。
*   **底视图 (趋势折线图)：** 引入时间维度（Year），展示 MPG 的均值漂移。这是系统的时间轴，也是次级控制器。

### 3.2 交互设计逻辑 (The "Why")
*   **刷选联动 (Brushing & Linking)：**
    *   *设计原因：* 为了实现“局部探索”。用户在散点图框选一部分车型，下方折线图会立刻重新计算这部分车型的历史趋势，右侧柱状图也会更新统计。这允许用户比较“高性能车”与“经济型车”的不同演化轨迹。
*   **双向高亮 (Bi-directional Selection)：**
    *   *设计原因：* 我在折线图中加入了 `yearClick` 参数。点击某一年的数据点，主散点图中该年份的车型会高亮，其余变淡。这实现了从“时间维度”反向回溯“性能分布”。
*   **一致性色彩映射：**
    *   全视图统一使用 `Origin` 字段驱动颜色，确保用户在切换视线时无需重新查看图例，降低认知负荷。

---

## 4. 开发细节与总结

*   **开发工具：** HTML5, Vega-Lite (v5), Vega-Embed, CSS3。
*   **技术挑战与突破：**
    *   最耗时的部分是**交互逻辑的闭环**。最初尝试实现多重过滤时，出现了交互冲突（例如框选后无法点击年份）。通过在 Vega-Lite 规格中使用 `params` 结合 `transform: filter` 数组，最终成功实现了“框选为主、年份点击为辅”的协同过滤逻辑。
    *   **坐标轴自适应：** 为柱状图设置了固定的 domain [0, 8]，防止在过滤时坐标轴剧烈跳动，保持了视觉的稳定性。

## 5. 如何运行

1.  **直接访问：** 点击上面的 [Live Demo 链接](https://4519645.github.io/-/)（推荐）。
2.  **本地运行：**
    *   下载本项目。
    *   确保网络连接正常（需加载 CDN 上的 Vega 核心库）。
    *   用浏览器打开 `index.html`。
3.  **交互操作指南：**
    *   **框选：** 在左上角散点图拖拽鼠标。
    *   **缩放：** 在散点图滚动鼠标滚轮。
    *   **过滤年份：** 在下方折线图点击具体的时间点（圆点）。
    *   **重置：** 双击图表空白处。
