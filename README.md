1. 你的可视化要回答的 问题 是什么？
本可视化旨在探索“20世纪70-80年代不同产地汽车的动力性能（马力）与燃油经济性（油耗）之间的权衡关系，以及不同产地车辆的硬件特征差异”。通过交互，我们希望帮助用户发现高马力汽车集中在哪些产地，它们的油耗表现如何，以及当我们聚焦于某一特定性能区间的车辆时，它们的平均气缸数有何特征。

2. 设计决策的原理

视觉编码：
我们选择了散点图作为主视图，因为散点图是展示两个连续定量变量（马力 vs 油耗）相关性的最佳选择。
使用颜色通道（名义型）来区分汽车产地（美国、欧洲、日本），以便快速识别区域集群。
右侧使用柱状图展示平均气缸数，柱状图能清晰地展示分类数据的聚合比较。
交互技术：
平移与缩放 （Pan & Zoom）： 数据点在某些区域较为密集，允许用户通过鼠标滚轮和拖拽来放大局部细节，避免视觉混乱。
按需详情 （Tooltips）： 悬停在散点上会显示车型、年份和具体数值，保持图面整洁的同时提供上下文信息。
刷选与多视图协同 （Brushing & Linked Views）： 这是本设计的核心亮点。我们在左图配置了二维区间刷选器（Interval Selection），右侧的柱状图会作为动态查询过滤器实时更新。这比简单的下拉菜单更直观，允许用户探索“高油耗低马力”或“高马力低油耗”特定子集的特征。
替代方案考虑：
最初考虑使用气泡图将“气缸数”编码为气泡大小，但发现重叠严重，导致“杂乱的设计”。最终决定采用多视图协同（散点图+柱状图），将维度拆分，这不仅降低了认知负荷，还满足了“有效多视图协同”的加分项要求。
3. 外部资源引用

数据来源： 使用了 Vega-Datasets 提供的开源 数据集（基于经典的普利茅斯汽车数据集）。cars.json
技术框架： 可视化基于 Vega-Lite （v5） 构建，并参考了 Vega-Lite 官方文档中的交互式多视图示例 （Interactive Multi-View Displays）。
